# 报价中倒计时功能完善总结

## 功能概述

本次完善了报价中页面的倒计时功能，包括倒计时显示、状态管理、数据刷新和用户体验优化。

## 主要改进

### 1. 倒计时显示逻辑优化

#### 新增方法：
- `shouldShowCountdown(row, priceType)` - 判断是否显示倒计时
  - 根据订单的最后竞争时间和报价间隔计算剩余时间
  - 支持不同价格级别的倒计时显示
  - 返回布尔值控制倒计时组件的显示

#### 功能特点：
- 支持最高价、高价、低价三个级别的倒计时
- 基于 `lastCompeteTime` 和 `quotationInterval` 计算剩余时间
- 智能判断倒计时是否应该显示

### 2. 倒计时组件优化

#### 显示格式改进：
- 统一显示格式：`剩余 X:XX 秒` 或 `剩余 XX 秒`
- 当时间小于1分钟时，只显示秒数
- 当时间大于1分钟时，显示 `分钟:秒` 格式

#### 样式状态管理：
- **正常状态** (`countdown-normal`): 蓝色主题，剩余时间充足
- **警告状态** (`countdown-warning`): 橙色主题，剩余时间少于5分钟
- **紧急状态** (`countdown-urgent`): 红色主题，剩余时间少于1分钟，带脉冲动画
- **已到期** (`countdown-expired`): 灰色主题，倒计时结束

### 3. 倒计时结束处理

#### 新增方法：
- `handleCountdownEnd(data)` - 处理倒计时结束事件
  - 防止重复通知机制
  - 自动刷新页面数据
  - 显示用户友好的提示消息
  - 自动清理通知记录

#### 功能特点：
- 使用 `Set` 数据结构防止重复通知
- 5分钟后自动清理通知记录
- 显示具体的订单号信息

### 4. 数据刷新机制

#### 定时刷新：
- `startCountdownRefresh()` - 启动30秒定时刷新
- `refreshCountdownData()` - 静默刷新倒计时数据
- `stopCountdownRefresh()` - 组件销毁时清理定时器

#### 生命周期管理：
- 组件挂载时启动定时器
- 组件销毁时清理定时器
- 避免内存泄漏

### 5. 用户体验优化

#### 视觉反馈：
- 不同剩余时间显示不同颜色
- 紧急状态添加脉冲动画
- 已到期状态显示灰色提示

#### 交互优化：
- 倒计时结束自动刷新数据
- 友好的提示消息
- 防止重复操作

## 技术实现

### 全局倒计时管理器
- 使用单例模式管理所有倒计时组件
- 统一的时间更新机制
- 自动清理已结束的倒计时

### 性能优化
- 全局定时器避免多个组件重复计时
- 静默数据刷新不影响用户操作
- 智能的通知防重复机制

### 样式设计
- 响应式设计，适配不同屏幕
- 平滑的过渡动画
- 清晰的状态区分

## 使用说明

1. **倒计时显示**：系统会根据订单的最后竞争时间和报价间隔自动显示倒计时
2. **状态提示**：不同颜色表示不同的紧急程度
3. **自动刷新**：倒计时结束时会自动刷新数据并显示提示
4. **防重复**：同一订单的倒计时结束只会通知一次

## 注意事项

1. 确保 `lastCompeteTime` 和 `quotationInterval` 字段正确传递
2. 倒计时基于客户端时间计算，可能存在时区差异
3. 建议定期同步服务器时间以确保准确性
4. 大量订单时注意性能影响，可考虑虚拟滚动优化
